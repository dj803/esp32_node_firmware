ESP32 Node Firmware + Node-RED — Deferred Improvements
======================================================
Generated: 2026-04-21

These items were surfaced by the v0.3.03 planning audit but intentionally left
out of the current release window. Keep this file as the backlog; revisit before
the next major version bump.

────────────────────────────────────────────────────────────────────────────────
FIRMWARE — SECURITY
────────────────────────────────────────────────────────────────────────────────

1. (shipped in v0.3.07) CSRF tokens on HTTP portals
   include/ap_portal.h: generateCsrfToken() uses esp_random() for 128-bit
   tokens. GET /  and GET /settings generate fresh tokens, embed them in
   hidden form fields, and set SameSite=Strict; HttpOnly cookies.
   POST /save and POST /settings verify the submitted field matches the
   server-stored token (single-use — invalidated after a successful check)
   and return HTTP 403 on failure.

2. (moved to WONT_DO.txt — decision not to change the hardcoded AP password)

3. (shipped in v0.3.11) HTTPS for provisioning portals
   include/ap_portal.h: WebServer replaced with ESP-IDF esp_https_server.
   Self-signed RSA-2048 cert generated on first boot (~10 s) via mbedTLS;
   stored in NVS namespace "esp32portal" and loaded on subsequent boots.
   CN = device UUID; IP SubjectAltName = 192.168.4.1 (fixed AP IP) so Chrome
   shows the "proceed anyway" option in AP mode. Both AP mode (port 443) and
   Settings mode (port 443 on STA interface) use the same cert. Plain HTTP
   fallback on port 80 if cert generation fails. MQTT cmd/config_mode status
   message now publishes https:// URL. Accepted UX cost: browser will warn
   "untrusted certificate" — click Advanced → Proceed to continue.

4. (shipped in v0.3.09) Non-blocking ESP-NOW channel scan

5. (shipped in v0.3.09) Non-blocking broker port scan
   include/broker_discovery.h: tryPortScan() rewritten with AsyncTCP parallel
   sliding window (PORTSCAN_WINDOW = 6). Worst case ≈ 7.6 s vs 19 s.

6. (shipped in v0.3.09) Remaining blocking delay()s in setup / loop
   setup() now returns immediately after WiFi.begin() or spawning the bootstrap
   task. All TODO(v0.3.07) comments removed. loop() drives the full boot state
   machine with millis()-based non-blocking Wi-Fi association.
   ap_portal.h delays deferred to v0.3.11 (HTTPS refactor touches same file).

FIRMWARE — CORRECTNESS
────────────────────────────────────────────────────────────────────────────────

9. (shipped in v0.3.10) Consolidate module-local state flags
   ap_portal.h: inline settingsServerIsRunning() accessor added; flag documented
   as strictly internal with a note that it must never be read from outside the
   module. espnow_responder.h: inline espnowResponderIsActive() accessor added.
   rfid.h: inline rfidIsReady() accessor added. No central bitmask — option (a)
   from the plan: keep flags module-local, enforce via accessors.

10. (shipped in v0.3.10) Per-peer rate limiting for ESP-NOW responder
    espnow_responder.h: token-bucket table (RATE_BUCKET_CAP=3, RATE_REFILL_MS=
    60000, RATE_MAC_SLOTS=8). Each credential request costs 1 token; refill 1
    per minute up to cap 3. New MACs start with CAP-1 tokens. Drop + LOG_W when
    tokens exhausted. LRU evict on full table via circular _rateBucketNext index.
    Upgraded from the existing 30-s flat cooldown that was already in the file.

11. (shipped in v0.3.08) Wire-format versioning for ESP-NOW WireBundle
    include/espnow_bootstrap.h: ESPNOW_WIRE_VERSION constant (=1) + uint8_t
    wire_version as first field of WireBundle (size 175→176 bytes). bundleToWire()
    sets wire_version=1; both receive paths check wire_version and LOG_W+reject
    frames with unrecognised values. SiblingHealth left unchanged (SIBLING_PRIMARY_
    SELECTION is disabled; SiblingHealth already carries protocol_version internally).

12. (shipped in v0.3.08) Heap churn in apHandleStatus
    include/ap_portal.h: apHandleStatus() now uses a 512-byte stack buffer and
    snprintf() instead of String concatenation.

13. (shipped in v0.3.08) NVS namespace registry
    include/config.h: enum class NvsNs : uint8_t (CREDENTIALS, DEVICE_ID,
    BROKER_CACHE, LED, RFID, BLE) + inline nvsNsName(NvsNs) lookup added after
    the last #define. All six existing #define constants kept unchanged as
    backward-compatible aliases.

14. (shipped in v0.3.08) Expanded unit test coverage
    include/semver.h: semverIsNewer() extracted from ota.h to a standalone
    dependency-free header; ota.h now includes semver.h.
    test/test_native/test_main.cpp: 6 semver tests (numeric patch comparison,
    double-digit minor, leading zeros, equal, major/minor ordering) + 4 additional
    pathological topic sanitizer tests (multiple slashes, leading slash, only
    wildcards, mixed). Total: 40 tests (was 30).

APPENDIX — 2026-04-21 SENIOR-DEV REVIEW (v0.3.12 / 0.3.13 / 0.3.14)
────────────────────────────────────────────────────────────────────────────────

A fresh senior-dev audit after v0.3.11 surfaced 23 findings across security,
robustness, and testability. They were grouped into three shippable packages:

Package A — v0.3.12 "Portal hardening" 🔴
  A1. XSS via unescaped HTML attribute interpolation (ap_portal.h).
      Fixed: htmlEscape() applied to every gAppConfig/DeviceId interpolation
      in both AP and Settings HTML builders.
  A2. Sibling-provided OTA URL writes to NVS without strict validation.
      Fixed: isSafeOtaUrl() in espnow_responder.h rejects non-http(s),
      over-length (>APPCFG_URL_MAX), or control-char inputs before NVS save.
  A3. CSRF single-global token race with httpd concurrent workers.
      Fixed: 2-slot token ring guarded by portMUX_TYPE; each /settings GET
      consumes a free slot; POST verifies and retires by token value.
  A4. Blocking mbedtls_rsa_gen_key watchdog risk (~10 s on first boot).
      Fixed: esp_task_wdt_reset() calls around the keygen + ctr_drbg_seed
      + pk_setup critical section in _generateTlsCreds.

Package B — v0.3.13 "Build reproducibility + network robustness" 🟡
  B1. Unpinned GitHub lib_deps (4 libraries).
      Fixed: pinned to exact commit SHAs (MFRC522v2, async-mqtt-client,
      AsyncTCP) and release tag (ESP32-OTA-Pull#v1.0.2) in platformio.ini.
      NimBLE-Arduino tightened from ^2.0.0 to ~2.0.0.
  B2. Task-WDT downgrade on OTA failure (async_tcp unsubscribed forever).
      Fixed: _otaReaddAsyncTcpToWdt() helper called on every failure path
      in otaCheckNow(); recursive-retry guard replaced with explicit
      isSiblingRetry parameter instead of function-static bool.
  B3. Rate-limit refill overflow smell (`tokens + raw` before clamp).
      Fixed: extracted to rate_limit.h as rateClampRefill(cur, elapsedMs)
      with pre-add headroom clamp; unit-tested in test_native.
  B4. Port-scan use-after-free risk (AsyncTCP callbacks fire after delete).
      Fixed: onConnect/onError/onDisconnect detached before close+delete.
  B5. RFID UID normalisation duplicated verbatim in add/remove handlers.
      Fixed: rfidNormalizeUid() helper in mqtt_client.h; both handlers
      call the shared helper.
  B6. String-concat JSON status builders (heap-fragmentation).
      Fixed: mqttPublishStatus and mqttPublishLedState use stack-allocated
      snprintf (512 B and 192 B buffers respectively).
  B7. NimBLE-Arduino silent 1.x downgrade risk.
      Fixed: #if !defined(NIMBLE_CPP_VERSION_MAJOR) || < 2 #error guard in
      ble.h; build fails loudly if someone reverts the lib pin.
  B8. Logging consistency — Serial.printf stragglers.
      Fixed: ~30 Serial.printf/println call sites in espnow_bootstrap.h
      migrated to LOG_D/I/W/E macros (completes the v0.3.06 migration).

Package C — v0.3.14 "Test coverage + observability" 🟢
  C1. semverIsNewer() silent-tolerant parser.
      Added: semverParse(s, maj, min, pat) strict overload — returns false
      on nullptr / malformed / missing-component / trailing-garbage / negative.
      Legacy tolerant API preserved for the OTA retry path.
  C2. WireBundle LE/pack contract undocumented.
      Added: doc comment in wire_bundle.h explaining the #pragma pack(1)
      requirement (LX6 inserts 7 B pad otherwise) and the LE-only endianness
      contract for timestamp. Struct extracted to include/wire_bundle.h so
      the on-wire contract is host-testable (176 B, version byte at offset 0,
      field round-trip via byte buffer).
  C3. bootstrap task stack size (8 KB) undocumented.
      Added: comment on espnowBootstrapBegin() citing the observed ~5.5 KB
      high-water mark, giving ~30% headroom.
  C4. MQTT plaintext-on-LAN design choice undocumented.
      Added: one-line comment in config.h next to BROKER_DISCOVERY_ENABLED
      cross-referencing SUGGESTED_IMPROVEMENTS.txt §7.
  C5. Test coverage thin on critical invariants.
      Added: 14 host tests in test/test_native/test_main.cpp
        • peer_tracker re-observe-when-full + partial expire (2)
        • rateClampRefill at cap / below cap / huge elapsed / zero (4)
        • WireBundle size / version-byte offset / byte round-trip (3)
        • ranging_math extreme RSSI finite / rssi > tx clamp (2)
        • semverParse accept / reject trailing garbage / reject missing (3)
      Total now ~54 tests (was 40).

  • MQTT-over-TLS to broker (§7) — private LAN; config.h notes the decision.
